小程序笔记-one

发表于 2018-03-06

微信小程序：
1-“不需要下载安装即可使用”：但严格意义是说不是真的就没有下载，小程序的安装包是控制在1M以下的，所以这个下载安装的过程是非常快的，以至于用户感受不到这个过程。
2-用户“用完既走”不用关心是否安装太多应用：且使用后没有痕迹残留等，做到迅速快捷 适用于低频的，偶尔的但是又可能是必须的产品
3-应用无处不在，随时可用：
B2C(人与商品)
P2P（人与人）
C2P（人与服务）

适合小程序平台的应用应该具有的特点：
1.业务逻辑简单
2.使用频率低但又是必要需求
3.性能要求低

传统 APP：体验差 成本高
小 程 序：流程简单 跨平台

也适合于简单的初创者，有想法但是却没有资源去实现，降低门槛，也会让市场变得多样化。

小程序也让微信变得像APPstore ，甚至接近一个操作系统。

对开发者的影响：
短期内将提升市场对JavaScript程序员的需求量
小程序是0基础开发者很好的入门平台
小程序不可以使用现在已经存在的JavaScript组件库(小程序不支持DOM，没有window对象，以DOM为基础操作的组件库，例如JQuery
以数据优先，不能操作DOM）
开发环境和开发逻辑极其简单，非常适合新手入门

Action:

网页编程采用的是 HTML + CSS + JS 这样的组合，其中 HTML 是用来描述当前这个页面的结构，CSS 用来描述页面的样子，JS 通常是用来处理这个页面和用户的交互。而在小程序里则对应的是WXML + WXSS + JS。

标签名字有点不一样 往往写 HTML 的时候，经常会用到的标签是 div, p, span，而小程序的 WXML 用的标签是 view, button, text 。
(其中有相似功能的是div–view span–text 还有最重要一点只有被text包裹的文本才能被用户执行长按选中)
在WEB中只要HTML中写好未出错的情况下便能被浏览器正确加载。而在小程序中必须在小程序的全局配置文件app.json中去指明，如：
如开发目录为：
pages/
pages/index/index.wxml
pages/index/index.js
pages/index/index.wxss
pages/logs/logs.wxml
pages/logs/logs.js
app.js
app.json
app.wxss
则需要在 app.json 中写
{
“pages”:[
“pages/index/index”,
“pages/logs/logs”
]
} 为了避免出错，且要在当前页面js文件中去引用page！
如： 当前页面的js文件为 index.js , 则需要早index.js文件中写
page({

})

（在app.json这个文件里了定义小程序的所有页面路径、界面表现、网络超时时间、底部 tab 等。）

关于images ：小程序里的px不是物理像素，而是针对显示器的逻辑像素，如果显示器的分辨率不同则物理像素跟逻辑像素的转换也会不同，所以小程序有自己的自适应像素RPX。

关于样式是用style 还是 写在样式表WXSS里：这里有个说法，从性能及复用性上来讲，静态样式应该写在样式表里，而动态样式则需要写在style里。
