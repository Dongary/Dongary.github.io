对于模板template： <template / >其实是占位符，他执行之后会消失，不会被打印，所以一些响应操作不能写在里边
(前端编码构建页面时：先静后动，先样式再数据)

部件在脱离文档流后经典居中方案：
.XXXX {
position: absolute;
left: 50%;
margin-left: -51rpx;

关于生命周期函数–监听页面加载：onLoad
一般会给一个参数options,这个参数里包含着从其他页面获取的值（前提有获取操作）。

对于一些用户操作过数据，并且在用户下次登陆时能直观查看其状态，这就涉及数据缓存。
获取缓存 wx.getStorge (key)
删除缓存 wx.removeStorge (key)
清除所有缓存 wx.clearStorge ( )
设置缓存 wx.setStorge (key,value)
以上每个步骤都分为同步与异步 StorageSync 和 Storge
需要注意的是:在小程序里如果用户不主动清除缓存则其会永久存在，且体积不能超过10M

在js语句里去引用data里的参数时参数名一定要加上this.data 列：this.data.参数名

JS经典错误：
深入理解this，this指的是函数调用的上下文环境，如果在自定义嵌套函数中使用this指针，像this.setData（{}）写法就不正确了，因为setData是属于Page（{}）中的data的。解决办法就是：在第一层自定义函数中的将this赋值给一个制定变量–var a = this，然后在嵌套函数中用a去引用this，即a.setData({})

同步和异步通常用来形容一次方法调用。

同步方法调用一旦开始，调用者必须等到方法调用返回后，才能继续后续的行为。
异步方法调用更像一个消息传递，一旦开始，方法调用就会立即返回，调用者就可以继续后续的操作。而，异步方法通常会在另外一个线程中，“真实”地执行着。整个过程，不会阻碍调用者的工作。

同步的缺陷是如果在这里执行不了，整个流程就会就此卡住，后面的代码走不了，会非常耗时；而在小程序中能用同步尽量用同步。

音乐播放：
wx.playBackgroundAudio(OBJECT) 使用后台播放器播放音乐，对于微信客户端来说，只能同时有一个后台音乐在播放。当用户离开小程序后，音乐将暂停播放；当用户点击“显示在聊天顶部”时，音乐不会暂停播放；当用户在其他小程序占用了音乐播放器，原有小程序内的音乐将停止播放。
wx.pauseBackgroundAudio()
暂停播放音乐。

乐播放在小程序上有个特点，你播放后如果跳转到另一个页面，音乐是不会被关闭的，（除非主动调用stop属性），其在页面会有一个总控开关。

音乐播放涉及的音乐源文件及封面图片都是不可以通过放在本地在调用来实现的，因为要控制体积（10M以下），都要采用网络链接的格式

因为音乐涉及版权问题，所以像qq音乐，网易云的外链都有保护措施，通常做法是用爬虫程序来获取MP3源，但是这个对于小白却不友好，当然大家也可以去尝试通过爬虫获取，这里也交给大家一个简单的步骤—http://up.mcyt.net/ 这个是可以提供可供使用外链的网站，但是资源不会那么全，有可能找不到想要的曲目，但是也能解燃眉之急。
