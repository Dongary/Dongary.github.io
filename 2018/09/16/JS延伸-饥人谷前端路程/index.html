变量提升：声明语句提前，赋值语句还是依照顺序来

作用域：如果在某个变量在函数语句中已经声明，访问的时候就是语句中声明的值；如果函数语句中没有声明，就会直接找（最父级下有没有这个变量）也就是window下的全局变量,不管函数在调用执行时有几层。

变量访问：访问的变量只是先访问到这个变量的.name属性，但是其值有可能发生变化（其赋值可能要比真正访问这个值要晚）

全局变量污染：（文件里要尽量少用全局变量，最好做到不用，因为存在值被覆盖的情况）
1：定义全局变量命名空间
只创建一个全局变量，并定义该变量为当前应用容器，把其他全局变量追加在该命名空间下。
2：利用匿名函数将脚本包裹起来
立即执行函数，通过定义一个匿名函数，创建了一个新的函数作用域，相当于创建了一个“私有”的命名空间，该命名空间的变量和方法，不会破坏污染全局的命名空间。

深拷贝与浅拷贝:
浅拷贝 —-只是拷贝了基本类型的数据，而引用类型数据，复制后也是会发生引用，我们把这种拷贝叫做“（浅复制）浅拷贝”，换句话说，浅复制仅仅是指向被复制的内存地址，如果原地址中对象被改变了，那么浅复制出来的对象也会相应改变。赋值操作不影响原来值
深拷贝 —-在计算机中开辟了一块新的内存地址用于存放复制的对象。赋值操作会影响原来值

promise:
在JavaScript的世界中，所有代码都是单线程执行的。
由于这个“缺陷”，导致JavaScript的所有网络操作，浏览器事件，都必须是异步执行。异步执行可以用回调函数实现：
AJAX就是典型的这种写法
在访问$.ajax(…).then(成功函数, 失败函数)
这种写法的好处在于，先统一执行AJAX逻辑，不关心如何处理结果，然后，根据结果是成功还是失败，在将来的某个时候调用success函数或fail函数。
那么如何自己生成 Promise 对象

setTimeout(()=>{
resolve() 或者 reject()
},3000)
})
}

MVC介绍：
MVC 是一种设计模式（或者软件架构），把系统分为三层：Model数据、View视图和Controller控制器。
Model 数据管理，包括数据逻辑、数据请求、数据存储等功能。前端 Model 主要负责 AJAX 请求或者 LocalStorage 存储
View 负责用户界面，前端 View 主要负责 HTML 渲染。
Controller 负责处理 View 的事件，并更新 Model；也负责监听 Model 的变化，并更新 View，Controller 控制其他的所有流程。
例如：移动端和PC端两个View，共享同一个Model
在MVC设计模式中， Model 响应用户请求并返回响应数据，
View 负责格式化数据并把它们呈现给用户，业务逻辑和表示层分离，
同一个 Model 可以被不同的 View 重用，所以大大提高了代码的可重用性。
